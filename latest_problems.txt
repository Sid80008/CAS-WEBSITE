## C. FIX RECOMMENDATIONS — Code Level

**Fix 1 — Login page crash (Frontend, Critical, 1–2 hours).** Open your browser console on `/login`. You will see a JavaScript error. The most likely cause is a missing `NEXTAUTH_SECRET` or `NEXTAUTH_URL` environment variable in your Vercel project settings. Go to Vercel → Project Settings → Environment Variables and ensure all auth-related variables are set. Add a top-level error boundary in your `_app.tsx` or `layout.tsx` so that if the login component crashes, it shows an error message instead of a white screen.

```tsx
// app/login/page.tsx — wrap in Suspense + error boundary
export default function LoginPage() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <LoginForm />
    </Suspense>
  )
}
```

Replace `+91 7457 244555` with `+91-7737689684`, replace `info@casanta.edu.in` and `info@centralacademyanta.com` with `centralacademyanta@gmail.com`, and replace all address variants with `Near Sahkari Petrol Pump, Kota Road, Anta, Baran, Rajasthan – 325202`.

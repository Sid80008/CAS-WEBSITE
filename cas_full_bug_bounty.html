🔐 Authentication & Session Bugs 5 bugs
B-01CRITICAL
Login redirects to homepage after 2–3 login/logout cycles instead of teacher dashboard
/admin/login → POST → should go to /admin/dashboard but goes to /
AuthPortal

Root cause: NextAuth's callbackUrl gets corrupted across multiple sessions. On first login it works. By the 2nd or 3rd cycle, the cookie holding callbackUrl is either stale, overwritten, or expired — so NextAuth falls back to the default redirect, which is /. This happens because NEXTAUTH_URL in Vercel env might not match exactly, or the redirect callback in auth.ts isn't hardcoding the role-based destination and instead trusting the dynamic callbackUrl which breaks.

Fix in lib/auth.ts callbacks: Replace the dynamic redirect with a hardcoded role-based one:
callbacks: {
  async redirect({ token }) {
    const role = token?.role
    if (role === 'ADMIN' || role === 'TEACHER' || role === 'OFFICE') return '/admin/dashboard'
    if (role === 'STUDENT') return '/portal/student/dashboard'
    if (role === 'PARENT') return '/portal/parent/dashboard'
    return '/'
  }
}

Also set pages: { signIn: '/admin/login' } in auth config so NextAuth always uses admin login as entry point for staff, not /.

🎨 Color Contrast & UI Bugs 4 bugs
B-15CRITICAL
Black text rgb(26,26,26) on dark navy bg rgb(0,34,70) — contrast ratio ~1.1:1, completely invisible
Fee page in parent portal, student portal fee section, any component using these two colors together
AccessibilityColor

Color rgb(26,26,26) = hex #1A1A1A (near-black). Color rgb(0,34,70) = hex #002246 (deep navy). Contrast ratio between these two is approximately 1.09:1 — almost identical luminosity despite being visually different hues. WCAG AA minimum is 4.5:1 for normal text. This text is effectively invisible. This is happening wherever a dark-text Tailwind class is applied inside a dark-blue card/box.

Global fix — find every instance of this combination:
Search codebase for #002246 or rgb(0,34,70) or [#002246]. Any text rendered inside these containers must use text-white or text-blue-100 or text-slate-100 — never text-gray-900, text-black, or text-[#1A1A1A]. Also fix bg-[#002246] containers in fee breakdown tables — all cell text must be white.
B-16HIGH
Same contrast problem spreading across other dark-bg portal sections
Anywhere dark text appears inside dark portal sidebar, dark card components
Accessibility

The contrast issue isn't isolated to the fee box. Any portal component that uses a dark background card (#002246, #001a3e, #0f172a, #1e3a5f) and then renders text with default dark Tailwind text classes will have this problem. This is a systemic theming issue — the dark portal theme was applied to backgrounds but the text colors weren't updated to compensate.

Fix globally: In your Tailwind config or global CSS, define a portal theme: any element with class .portal-card or .dark-section should force color: white and color-scheme: dark on all children. Audit every portal page with browser DevTools Accessibility panel → Color Contrast checker.
